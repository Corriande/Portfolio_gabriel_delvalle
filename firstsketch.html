What is a Coding Language?

Python is one of many coding languages. Just like human languages, between coding languages there are many different ways to say the same core idea. For example, in human languages we have many ways to say "Hello":
Hallo
Hola
Bonjour
你好
こんにちは
नमस्ते

In this case the core meaning being expressed in different ways is a greeting. In coding, the core meaning which stays the same between different coding languages is the logic and math which accomplishes our goal. The way we express the logic and math using the language is the syntax, which is similar to the vocabulary and the grammar of a human language. We need to know how to accurately translate our ideas into syntax for the computer to understand our message.

Like learning a language, learning proper syntax takes memorization and practice. However, Python was designed to have a very simple and intuitive language. There's a few simple rules that student's catch on to quickly after following along, typing code line by line with the instructor. Instead of syntax, most of the challenge of coding is in accomplishing a goal or expressing an idea using the mechanisms of logic and math. This creative and problem solving skill is relevant to all coding languages. An idea when clearly expressed as logic and math can be easily translated to syntax. For example, if you can express an idea logically, a Large Language Model such as ChatGPT can often accurately translate the idea into syntax for you.

Thus, these lessons emphasize the logical and creative ideas beneath the code itself.

What makes Python Special?

Python is known as the coding language for 10 year olds and PHD's. It was designed to make powerful coding techniques easily accessible, with simple syntax and fewer computational details to consider. Python is a High Level Language which actually means it is simpler, closer to how humans think, but also slower. Low Level Languages like C, C++, Rust, or Go are closer to the 1's and 0's computers calculate with at the most fundamental level, so they are more difficult to use, but also faster.

Some coding languages are built for specific purposes, like JavaScript which is used for web development.

Python can also be used for web development, as well as apps, controlling electronics and robots, creating media and video games, and especially, working with data. Python is a skill valuable to professions across STEM, finance, research, and design for its ability to manipulate data. Python's wide use in working with data has made it a core tool in the creation and use of AI tools. For creative purposes Python can be used inside the free but powerful 3D modelling software Blender, and many professional game engines such as Unreal Engine and Unity have the option to code in Python.

RGB colors

We can create any color on a computer with a combination of Red, Green, and Blue.

"But wait!" you might think, "How do you make yellow using red, green, and blue?"

It's true that if you were using paint no combination of red, green, and blue would ever give you yellow. But instead of paint we're using pixels, tiny squares of light on your screen that are each part red, green, and blue. Each 3 rectangle grouping of red, green, and blue in the image below is a single pixel.

Since each value of RGB is at its max value (255) in the image below color(255, 255, 255) from afar your eye would see the image below as pure white.

So how do we make yellow?
You lie to the eye using red and green.
Our eyes have light receptors for red, green, and blue, which interpret colors such as yellow when both the red and green receptors are stimulated.

Whenever you think you see yellow on a computer screen, this is what is really there.
color(255, 255, 0)

When each RGB value is the same, for example color(100, 100, 100), the color is on the greyscale. If we input a single number as a color, processing will interpret it as greyscale:
color(0) gives black
color(100) gives grey
color(200) gives light grey
color(255) gives white

First Animation: Sunrise Sunset

In the first sketch, even if nothing was moving, as long as the code was being run (if you hit the play button and didn't stop the code afterwards) the canvas was being refreshed many times per second, drawing the shapes in the same positions each time the draw() function looped. To create movement, all we have to do is have the position of one of the shapes change between each loop of the draw() function.

Let's start by drawing a circle like a sun on a blue background and making it go down.

In the code above, move_y += 1 increases the value of move_y by 1 on each loop of the draw function, and by adding move_y to the Y position of our circle it will be drawn lower on the canvas with each loop, creating a smooth descent. If we wanted the sun to descend faster we could simply increase the amount added on each loop, for example, move_y += 5.

Now here's an important nuance. Why did we define move_y = 0 outside the draw function? Remember that as the draw function loops each command inside of it will be called from top to bottom. If we start the draw function with move_y = 0, and then we try to increase its value at the end of the loop with move_y += 1, when the loop completes and we return to the top of the draw function for another cycle, move_y = 0 would be called again, erasing any change over time we hoped to create.

Thus, we define move_y = 0 in the global space (outside any function) where it will only be called once when the code is first run. However, when we define a function in Python, it assumes each variable inside of it is unique unless explicitly told otherwise. This is why we've added to the draw function global move_y, which specifies that the move_y variable in the draw function is the same variable in the global space. We need to add this to any function where we intend to change the value of move_y, but it is only necessary to add to a function if you intend to change the value of the variable in that function.

Now the circle moves down, but it also disappears below the canvas, falling forever. Now what?

To create an animation where the sun rises and sets, we want the sun to reverse direction and start rising up once it passes the bottom of the screen. Let's break this goal down into two logical challenges:

1. How do we know when the sun goes below the canvas?

When move_y = 0, the Y position of the sun is 250, the center of the canvas.

The distance between the center of the canvas and the bottom is 250.

Thus, when move_y = 250, the sun is halfway off the screen, the center of the sun positioned at the bottom edge.

Since the radius of the sun is 50, we know the sun will be fully off the screen when move_y = 300, and we can add a little more to that so there's a brief pause between its disappearance and it rising again.

2. How do we make the sun rise instead of descend?

Since move_y += 1 made the sun move down, it should be simple to recognize that the opposite of this, subtracting one on each loop with move_y -= 1 what make the sun move up.

However, we cannot change += to -= in the middle of our code running. What we can do is replace the static number 1 with a variable such as pos_min that can have its value changed whenever we please to 1 or -1, allowing us to increase or decrease the value of move_y with move_y += pos_min

Set a condition with if:

"If you listen, then you will learn".

This is an exmaple of a conditional statement I provide to students. We know in this statement that "you will learn" will only happen "If you listen".

if-statements let us specify when we want something to happen. In this case we want the sun to switch directions to rise when it reaches below the canvas and switch directions to descend when it reaches above the canvas. We know that we can do this by changing pos_min = -1 and pos_min = 1 respectively. So let's bring it all together:

Notice the spacing here!

Code is moved forward to be inside each if-statement.

But each if-statement is aligned to the one above it. If the second if statement were moved forwards to be aligned with pos_min = -1, the second if-statement would be inside the first one and could only activate if the first one were also activated!

With these if-statements the sun will now rise up and down. Now let's add a final detail and fade the sky from blue to pink. The red value of our background is 0. To turn the sky pink we just need to increase the red value, and we want this to happen gradually as the sun is moving down but not as it moves up.

Luckily we already have a variable which increases as the sun moves down and decreases as the sun moves up: move_y. Thus to complete your first animation, change the background color with:

background(move_y, 200, 250)

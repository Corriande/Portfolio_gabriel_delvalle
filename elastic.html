Elastic Introduces Object Oriented Programming

Many kinds of coding projects require us to make several copies of the same code which behave the same but store and act on different data values. For example, user accounts on a website, or enemies in a video game.

Each account a users on a website create are distinct entities that behave the same but store different values.

Each creeper in Minecraft behaves the same but has a different location and different states of action, such as wandering or attacking the player, which can be turned on or off.

In both of these cases, we want the code to be able to generate a new user account or new creeper on its own when the situation calls for it.

Object Oriented Programming (OOP) allows us to structure our code to efficiently create intricate projects with many moving parts and unique entities. Python is one of many Object Oriented coding languages. Others include C++, C#, Java, JavaScript, Swift, and R.

An example of a language that is not Object Oriented is C, because it is a very low level language, not meaning it is easier or less useful (quite the contrary) but rather that it is structured closer to how computers operate rather than what is convenient to humans.

Objects are the unique entities we generate that can store their own unique values and can execute their own unique actions. A Class is a template for an object, where we define what variables and functions all the objects made from that class will have.

Notice in the image above how when we are coding a class, the variables or functions that an object will inherit are denoted by the prefix self. and we access these inherited functions and variables for each object using the prefix object_name.

But we don't want the circle to follow our mouse perfectly. We want it to bounce with elasticity in response the mouse movements, like the circle were attached to the mouse by a rubber band.

This project introduces the concept of coding velocity, which is simple but useful for the creation of all kinds of moving objects.

Velocity represents both speed and direction.

OOP allows us to give each object its own values for velocity on the X axis and velocity on the Y axis (In 2D).
We move objects in 2D by updating their X and Y positions with their velocities. For example, if velocity X is vx:

obj.vx = 5
obj.x += obj.vx
This will cause obj to move to the right because we are increaseing the X position of obj.

If the value of obj.vx were 1 it would still move to the right but slower.

If the value of obj.vx were -1 obj would move to the left.

If we want the object to reverse its direction, like to bounce off of a wall, we can simply reverse the velocity:

obj.vx *= -1

With this change the object will now move to the left at the same speed it was moving to the right because now obj.vx equals -5.

*= is like += but instead of adding a value to a variable it multiplies the value of a variable.

The interactive code below demonstrates how the simple physics will work before we structure it with OOP:

Subtracting mouse.x - x gives us both the distance between the mouse and the circle on the X axis (the absolute value) and the direction the circle must travel to reach the mouse based on the sign of the value (positive or negative).

For example, if our circle is on the right side of the screen with an X of 300 and the mouse is on the left side of the screen with an X of 100: dx = 100 - 300 which gives us -200.
Based on the negative sign we know the circle must travel to the left to reach the mouse.

vx += dx * 0.1 adds a tenth of the distance to vx.
If vx was positive before because the circle was following the mouse to the right, adding this negative value will decrease the velocity, slowing the circle's rightward movement, until vx is 0 (unmoving), and then grows in the negative direction, causing the circle to bounce towards the mouse.

Whenver we move the mouse, acceleration occurs as we are increasing the rate of speed based on the distance with vx += dx * 0.1. As the circle grows closer to the mouse it decelerates. This is partly because dx grows smaller as the circle approaches the mouse, but this alone is not enough to shrink the velocity to 0. If vx = -20 and dx = 0 (distance = 0) the circle will overshoot the mouse and keep moving to the left. When the circle overshoots the mouse moving to the left, the distance in the positive direction will continue to grow, causing the circle to spring back to the right.

To see this effect, add a hash tag before the two following lines of code in the interactive code above to disable them and see how the circle is unable to stop moving, able only to accelerate, springing forwards and backwards around the mouse even if the mouse is not moving:

# vx *= elastic
# vy *= elastic

It's these two lines of code above which allow the circle to decelerate and eventually reach the mouse, by dampening the velocity of the circle, shrinking the velocities towards 0 at a rate determined by the value of elastic, which for this effect can be any value less than 1 and greater than 0.

The greater the value of elastic, the more the circle will bounce and swing. This is because an elastic value of 0.5 will decelerate the velocity to 0 faster than a value of 0.9 using vx *= elastic.

Try changing the value of elastic in the interactive code above and see what different values do.

When defining the objects in the code above, if we were to not include the inputs when initializing the objects by leaving the parentheses empty, then we would get an error like so:

When we defined __init__(self, elastic, color) we specified that the function expects two inputs, elastic and color. The function won't execute if not given the number of inputs or in other words arguments specified.

We can also see from the error that reads __init__() takes atleast 3 arguments (1 given) that even though self is an expected argument, we don't need to input it when generating an object. This is because self represents the object itself. Notice how self is also input into to the functions dispay(self) and move(self, mx, my). This allows these functions to access the values of variables like self.x which were defined in __init__.

This is the key to why we redefine the values we input when initializing the function like so:
self.elastic = elastic
While elastic, as an input to __init__, is only available to the __init__ function, self.elastic or any value with the prefix self. can be accessed by any function in the class with self as an argument.

A fun note on boolean variables:
It's common knowledge that computers work with electricity and make calculations or store information in binary with 0's and 1's. A boolean variable is the most basic kind of information a computer can store. Its value can be either True or False. In other words, 1 or 0, or if you think of a light switch, On or Off. A transistor is like a tiny light switch that stores information and performs calculations inside a computer. Modern computers have many billions of transistors. Transistors either have electricity flowing in them or not, in other words, they are either On or Off, 1 or 0, True or False.
